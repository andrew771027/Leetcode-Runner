# v5.0 — Introduce YAML Configuration

## 目標

先把 Runner 的設定從程式碼中抽出來，放到 `runner.yaml`。

這一版還不是完整的 config-driven architecture，而是開始建立「外部設定檔」的概念。

## 為什麼會有這一版

v4.9 之後，Runner 已經有 Pipeline、Stage、Context、Backend 等概念。

但目前許多參數仍然寫在程式碼裡，例如：

- 要跑哪個 repo
- 使用哪個 backend
- artifact 輸出位置
- reporter 類型

每次要調整設定，都需要改 code。

所以 v5.0 的第一步是：

> 先建立 runner.yaml，讓設定可以被外部描述。

## runner.yaml 範例

```yaml
runner:
  name: leetcode-runner
  version: v5.0

execution:
  backend: subprocess
  repo_path: ../leetcode
  test_path: tests/

artifact:
  output_dir: artifacts/

report:
  type: console
